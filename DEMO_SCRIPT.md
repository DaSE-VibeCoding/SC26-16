# CampusMate 演示脚本

## 演示前准备

1. 启动项目并访问首页。如果之前做过演示，进入“个人中心”点击“恢复演示数据”。
2. 演示账号为“林小羽”“周同学”“陈同学”，密码均为 `123456`。
3. 所有数据只保存在当前浏览器的 `localStorage` 中。

## 五分钟核心路径

1. 在发现页浏览活动，使用关键词、分类、日期和地点筛选。
2. 打开活动详情，展示时间、地点、人数、发起人、成员、收藏、评论和举报入口。
3. 使用“林小羽”登录，创建一条明天开始的活动，确认活动进入详情页和“我的组队”。
4. 切换到另一个演示账号，申请加入刚创建的活动。
5. 切回发起人账号，在“我的组队 → 待我审批”同意申请。
6. 打开通知中心，点击通知进入对应活动，并演示删除通知。
7. 进入个人中心，编辑头像、昵称、学院、年级、简介和兴趣标签。
8. 展示个人统计、活动历史、成就徽章、信用分规则说明和完成评价摘要。
9. 打开“通知与隐私设置”，修改通知偏好、公开状态、申请时可见资料、活动偏好和时间偏好，并演示退出登录入口。
10. 打开“管理员演示视角”，展示数据概览、活动审核下架/恢复和举报处理。

## 可选增强演示

- 活动结束后，成员之间提交完成评价，观察信用分和评价摘要变化。
- 在个人中心导出 JSON，说明本地数据结构；再用“恢复演示数据”一键回到初始状态。
- 上传头像图片或选择系统头像，说明头像数据仅保存在当前浏览器。

## 交付前检查

```bash
npm.cmd run lint
npm.cmd run test:e2e
npm.cmd run build
```

## 已知边界

- 这是课堂展示型 MVP，不包含真实身份认证、云数据库、多设备同步、支付、地图或实时聊天。
- 演示账号快捷登录按钮不是必需项；当前通过展示账号和统一密码来保留完整登录流程。
- 管理员视角是本地演示功能，不代表正式后台权限系统。
