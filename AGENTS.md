# CampusMate Codex 开发规则

## 项目目标

CampusMate 是校园搭子匹配平台的课程展示型 MVP。优先保证可重复演示，不为了正式运营引入云数据库、真实认证、支付、地图或实时聊天。

## 开发顺序

1. 阅读 `README.md`、`CampusMate_完整企划书.md` 和当前 Issue。
2. 先说明计划和将要修改的文件，再开始修改。
3. 一次只实现一个 Issue 的范围，不顺手扩展无关功能。
4. 复用现有 `AppProvider`、`lib/types.ts` 和 `lib/storage.ts`，不要复制新的状态系统。
5. 完成后运行 `npm run lint` 和 `npm run build`。
6. 在 PR 描述中记录修改内容、验证命令和未完成事项。

## 模块边界

- 页面只负责渲染、表单输入和路由跳转。
- 用户、活动、申请和通知的状态操作集中放在 `components/app-provider.tsx`。
- 本地持久化集中放在 `lib/storage.ts`。
- 类型定义集中放在 `lib/types.ts`。
- 初始演示数据集中放在 `lib/seed.ts`。
- 公共样式优先使用现有 Tailwind 组件类。

## 重要约束

- 不要直接向 `main` 推送。
- 不要把密钥、真实手机号或真实账号写入仓库。
- 不要修改其他成员负责的模块，除非 Issue 明确要求或 PR 中说明原因。
- 不要为了通过检查删除现有功能。
- 修改公共接口时，要说明影响到哪些页面。
- 如果需求不明确，先提出最小可行实现和验收标准。
