# D 模块交付说明

## 交付范围

D：用户、数据与质量模块已完成必做功能，并补齐了为 D 闭环所需的部分 B/C 缺口。

## 完成内容

- 登录页、注册页、演示账号说明。
- 个人资料页：头像上传、系统头像、昵称、学院、年级、个人简介、兴趣标签。
- 用户偏好：可接受活动类型、时间段偏好。
- 隐私设置：个人主页公开 / 私密、申请活动时发起人可见字段配置。
- 数据统计：参加活动数、发起活动数、完成率、信用分。
- 活动历史、个人成就 / 徽章。
- 信用分规则说明，默认折叠。
- 活动完成后的成员评价体系。
- 通知中心：通知偏好、活动通知跳转、通知删除、全部已读。
- 本地数据模型、统一状态管理、Mock 数据初始化、本地迁移。
- 数据导出 JSON、导入 JSON、恢复演示数据。
- 全局错误兜底页和 404 页面。
- 简易管理员视角：数据概览、活动审核、活动下架 / 恢复、举报处理。
- Playwright 关键流程测试。
- README、演示脚本、功能清单维护。

## 不作为必做项的说明

“演示账号快捷登录按钮”未实现。理由：登录页已经展示演示账号和统一密码，手动输入可以保留完整登录流程；除非验收方明确要求一键演示，否则该项不是必要功能。

## 已知边界

- 所有数据仅保存在当前浏览器 `localStorage`，不支持多设备同步。
- 管理员视角仅用于课堂演示，不代表正式权限系统。
- 评价、信用分和通知均为本地演示规则，不代表真实运营规则。
- 当前代码尚未提交到 Git 分支或创建 PR。
