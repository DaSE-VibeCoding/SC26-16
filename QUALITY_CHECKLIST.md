# D 模块功能清单与质量回归

在无痕窗口或恢复演示数据后执行，并在每次集成前复测。

## 必做功能

- [x] 登录页与注册页可用。
- [x] 登录页展示演示账号和统一密码。
- [x] 个人资料页支持头像、昵称、学院、年级、个人简介编辑。
- [x] 头像支持上传图片、系统头像和首字母回退。
- [x] 兴趣标签可编辑。
- [x] 设置页支持可接受活动类型和时间段偏好。
- [x] 个人中心展示参加活动数、发起活动数、完成率、信用分。
- [x] 个人中心展示活动历史记录。
- [x] 个人中心展示个人成就或徽章。
- [x] 设置页支持通知偏好、隐私开关和退出登录。
- [x] 使用 `lib/types.ts`、`lib/storage.ts`、`AppProvider` 做统一数据模型和状态管理。
- [x] Mock 数据初始化和“恢复演示数据”可用。
- [x] 全局错误兜底页和 404 页面可用。
- [x] Playwright 覆盖创建活动、设置保存、通知删除等关键状态变化。
- [x] README、演示脚本和功能清单已维护。

- [x] 数据导出为 JSON。
- [x] 一键恢复演示环境。

## 手动回归路径

- [ ] 使用“林小羽 / 123456”登录，刷新后仍处于登录状态。
- [ ] 注册唯一昵称；再次注册同名昵称时收到中文错误提示。
- [ ] 在个人中心编辑头像、简介和兴趣，刷新后确认资料仍保留。
- [ ] 在设置页保存活动偏好、时间偏好、通知开关和隐私开关，刷新后确认仍保留。
- [ ] 点击“退出登录”，确认回到登录页。
- [ ] 点击“恢复演示数据”，确认新建账号、活动、申请和通知被清除。
- [ ] 创建活动、申请加入、审批通过、通知跳转活动页。
- [ ] 删除一条通知，刷新后确认不再出现。
- [ ] 提交活动举报，在管理员页处理举报。
- [ ] 在管理员页下架并恢复一条活动。
- [ ] 活动结束后提交成员评价，确认信用分和评价摘要更新。
